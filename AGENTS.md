## Project Overview

**WhatSaaS** is a multi-tenant WhatsApp Business management SaaS platform. It enables teams to manage WhatsApp communications, automate conversations with flow builders, handle AI-powered responses, run campaigns, manage contacts, and integrate voice calling.

## Tech Stack

- **Framework**: Next.js 16.1.1 (App Router, Turbopack)
- **Language**: TypeScript
- **Database**: PostgreSQL with Drizzle ORM
- **Payments**: Stripe (Razorpay also integrated)
- **UI**: shadcn/ui (Radix UI + Tailwind CSS 4.1.7)
- **Auth**: Custom JWT via `jose` library
- **Real-time**: Pusher
- **AI**: OpenAI, Google GenAI
- **WhatsApp**: Evolution API integration
- **Voice**: Twilio
- **i18n**: next-intl (en, pt, es)

## Common Commands

```bash
pnpm install          # Install dependencies
pnpm dev             # Start dev server (Turbopack)
pnpm build           # Production build
pnpm start           # Start production server

# Database
pnpm db:setup        # Create .env from template
pnpm db:generate     # Generate Drizzle migrations
pnpm db:migrate      # Run migrations
pnpm db:seed         # Seed test user (test@test.com / admin123)
pnpm db:studio       # Open Drizzle Studio

# Testing
pnpm test            # Run tests (Vitest)
pnpm test:watch      # Watch mode

# Stripe (for local webhook testing)
stripe listen --forward-to localhost:3000/api/stripe/webhook
```

## Architecture

### Multi-Tenancy Model

Data isolation is via `teamId` on most tables. Core entities:
- `users` - authentication (email/password, JWT sessions)
- `teams` - organizations with stripeCustomerId, stripeSubscriptionId, planId
- `teamMembers` - user-team association with roles (owner, admin, agent)
- `plans` - subscription plans with feature flags (maxUsers, maxContacts, isAiEnabled, etc.)

### Route Groups

- `app/[locale]/(dashboard)/` - Protected dashboard routes (require session)
- `app/[locale]/(login)/` - Auth routes (sign-in, sign-up, password reset)
- `app/api/` - API routes, most require `teamId` context from session

### Key Library Modules

- `lib/auth/` - Session management, API key auth, permission guards
- `lib/db/` - Drizzle schema and database utilities
- `lib/payments/` - Stripe integration, checkout, webhooks
- `lib/automation/` - Flow builder engine and execution
- `lib/ai/` - AI provider configs and sessions
- `lib/chat/` - Chat handling, message processing
- `lib/rate-limit.ts` - Rate limiting middleware

### Permission System

Roles: `owner`, `admin`, `agent`
Granular permissions: `automation`, `aiAgent`, `contacts`, `templates`, `campaigns`, `voiceCalls`, `settings`
Chat visibility: `all`, `assigned`, `department`

## SaaS Readiness Status

### Implemented
- Multi-tenancy via teams + teamMembers
- Stripe subscription billing (checkout, portal, webhooks)
- Role-based access control with permission guards
- Plan-based feature flags
- Team-scoped data isolation
- Rate limiting
- Invitation system
- Activity logging
- Email verification on sign-up
- OAuth providers (Google, GitHub)
- Two-factor authentication (2FA) with TOTP
